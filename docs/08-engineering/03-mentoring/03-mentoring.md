# 带新人：技术传承与知识管理

> 考察频率：★★☆☆☆  难度：★★★☆☆  
> 高级工程师必备：技术传承能力

## 核心问题

带新人考察什么：
1. **知识表达能力**：能不能把经验讲清楚
2. **因材施教**：不同人不同带法
3. **知识沉淀**：能不能把经验留下来

## 1. 新人培养路径

### 第一周：熟悉期

```markdown
## Day 1-2：环境搭建
- 开发环境配置
- 代码仓库权限申请
- 本地服务启动
- 文档阅读（架构图、接口文档）

## Day 3-4：代码走读
- 核心模块代码讲解（1v1）
- 提一个小 bug 修复任务
- 熟悉 code review 流程

## Day 5：第一个任务
- 分配一个简单任务（如：增加一个接口字段）
- 讲解任务背景和预期
- 当天 code review 当天合并
```

### 第一月：成长期

```markdown
## Week 1-2：独立完成任务
- 分配小块任务（1-3 天）
- 每天站会同步进度
- 遇到问题及时辅导

## Week 3-4：独立排查问题
- 分配线上问题排查
- 讲解排查工具和方法
- 总结复盘写成文档
```

## 2. 代码走读方法

### 新人代码 review 重点

```go
// 代码走读不要只说"这里不好"，要讲为什么
// ❌ 不好：这个变量命名不好
// ✅ 好：这个变量叫 userID，但存的是 userUUID，容易让人误解，建议改名或加注释

// 代码走读 checklist
// 1. 命名是否清晰
// 2. 错误是否处理
// 3. 边界条件是否考虑
// 4. 是否有重复代码可以抽象
// 5. 注释是否必要
```

### Go 代码 review 示例

```go
// ❌ 新人写的代码
func getUser(id int) *User {
    u := db.Query("select * from users where id=?", id)
    return u
}

// ✅ Review 后
func getUser(ctx context.Context, db *sql.DB, id int64) (*User, error) {
    // 1. 参数校验
    if id <= 0 {
        return nil, errors.New("invalid user id")
    }
    
    // 2. 带上 context
    // 3. 错误处理
    // 4. 扫描到 struct
    var user User
    err := db.QueryRowContext(ctx, 
        "SELECT id, name, email FROM users WHERE id = ?", id,
    ).Scan(&user.ID, &user.Name, &user.Email)
    if err != nil {
        if err == sql.ErrNoRows {
            return nil, nil  // or errors.New("user not found")
        }
        return nil, fmt.Errorf("query user: %w", err)
    }
    return &user, nil
}
```

## 3. 技术文档怎么写

### 好文档的标准

```markdown
# API 文档示例

## 接口名称
获取用户信息

## 请求
GET /api/v1/users/:id

## 参数
| 名称 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int64 | 是 | 用户ID |

## 响应
```json
{
  "code": 0,
  "data": {
    "id": 1,
    "name": "张三",
    "email": "zhangsan@example.com"
  }
}
```

## 错误码
| code | 说明 |
|------|------|
| 10001 | 用户不存在 |
| 10002 | 权限不足 |

## 示例
```bash
curl -X GET http://api.example.com/api/v1/users/1
```
```

## 4. 知识分享怎么做

### 内部技术分享

```markdown
# 技术分享：如何做性能优化

## 1. 分享目标
让团队掌握性能优化的方法论，而不是只讲一次案例

## 2. 准备清单
- [ ] 清晰的标题和目标
- [ ] 3 个以内的核心案例
- [ ] 可运行的代码示例

## 3. 分享流程（30 分钟）
- 5 分钟：问题背景
- 10 分钟：排查过程
- 10 分钟：解决方案
- 5 分钟：总结和 Q&A

## 4. 后续
- 整理成文档
- 代码示例上传仓库
- 沉淀为团队知识库
```

## 5. 面试话术

**Q：你是怎么带新人的？**

> 我会分阶段：前两周以熟悉为主，每天 code review，当天的 bug 当天修；后两周给独立任务，遇到问题先让新人自己思考，我会给方向性的提示而不是直接给答案。核心是培养独立解决问题的能力，而不是让他依赖我。

**Q：怎么建立团队知识库？**

> 1）文档先行：新功能必须有文档才能提测；2）周报沉淀：每周技术问题整理成 wiki；3）代码即文档：代码里的注释和 README 要写清楚；4）复盘机制：每个项目结束必须写复盘文档。

**Q：代码规范怎么推广？**

> 不能靠口头要求，要靠工具落地。我的做法是：1）CI 集成 golangci-lint，lint 不通过不能合并；2）code review 时重点检查规范问题；3）新人第一次犯错只提醒，第二次写进规范文档，第三次再扣绩效。
