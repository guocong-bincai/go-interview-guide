# 面试高频场景题（开放性问题）

> 考察频率：★★★★★  优先级：P0

## TODO（待填写）

> 这类题没有标准答案，考察系统性思维和表达能力。每道题给出「答题框架」和「加分点」。

## 1. 如何设计一个高可用的系统
- [ ] 答题框架：冗余（多副本）→ 隔离（故障域）→ 限流降级（自我保护）→ 监控告警（快速发现）→ 快速恢复（自动摘流）
- [ ] 加分点：量化 SLA（几个9）、混沌工程验证

## 2. 你们系统最大的技术债是什么，怎么还的
- [ ] 答题框架：识别（症状）→ 评估（影响范围/优先级）→ 方案（渐进式 vs 重写）→ 落地（里程碑/灰度）→ 结果（量化改善）
- [ ] 加分点：讲清楚为什么当时会产生这个债

## 3. 如果让你重新设计现在的系统，你会怎么做
- [ ] 答题框架：先说当前的问题是什么 → 再说重新设计的思路 → 然后说风险和权衡
- [ ] 加分点：不要全盘否定现有系统，说明当时的决策背景

## 4. 线上出了 P0 故障，你怎么处理
- [ ] 答题框架：止损（切流/回滚/降级）→ 定位（日志/监控/链路追踪）→ 修复（最小改动）→ 恢复（验证/逐步放量）→ 复盘（根因/改进项）
- [ ] 加分点：用具体的真实故障案例

## 5. 你负责的系统 QPS 从 1000 提升到 10 万，怎么做
- [ ] 答题框架：找瓶颈（Profiling）→ 横向扩展（无状态服务加机器）→ 纵向优化（缓存/异步/DB优化）→ 架构升级（读写分离/分库分表）
- [ ] 加分点：每步给出量化指标
